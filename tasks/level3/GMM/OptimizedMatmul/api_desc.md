# aclnnOptimizedMatmul

## 功能描述

### 算子功能
该Ascend C算子用于执行两个二维矩阵的高性能乘法运算，即 `A` 乘以 `B`。与基础矩阵乘法不同，该算子通过内部数据填充（Padding）和内存布局优化策略，针对昇腾硬件架构进行了深度优化，旨在最大化计算效率和内存带宽利用率，尤其是在处理特定维度的大型矩阵时。它是深度学习模型中各种计算密集型层（如全连接层、注意力机制）的关键组成部分。

### 计算公式
假设输入张量为 $A$（维度为 $m \times k$）和 $B$（维度为 $k \times n$），则输出张量 $C$（维度为 $m \times n$）的计算公式如下：

$$C_{ij} = \sum_{p=1}^{k} A_{ip} B_{pj}$$

其中，$i$ 的范围是从 $1$ 到 $m$，$j$ 的范围是从 $1$ 到 $n$。$C_{ij}$ 表示输出矩阵 $C$ 中第 $i$ 行第 $j$ 列的元素值。

### 计算过程与类型转换
为了在执行大规模累加操作时保持较高的数值精度并防止数据溢出，该算子在内部计算过程中采用了高精度累加的策略。具体流程如下：

1.  算子接收两个数据类型为 `float16` 的输入张量 `a` 和 `b`。
2.  在执行乘加计算之前，内部会将 `float16` 类型的输入数据转换为 `float32` 类型。
3.  所有的乘法和累加计算均在 `float32` 精度下完成，以确保结果的准确性。
4.  所有计算完成后，得到 `float32` 类型的结果矩阵。
5.  最后，将 `float32` 的结果张量转换回 `float16` 类型，作为算子的最终输出。

## 接口定义

### 算子原型定义接口
#### Input
- a：Device侧的aclTensor，公式中的A，数据类型支持float16，维度支持2维，数据格式支持ND。
- b：Device侧的aclTensor，公式中的B，数据类型支持float16，维度支持2维，数据格式支持ND。
#### Output
- c：Device侧的aclTensor，公式中的C，数据类型支持float16，维度支持2维，数据格式支持ND。
#### Attr
- 无

## 约束与限制
  * 输入张量 `a` 和 `b` 的数据类型当前仅支持 `float16`。
  * 输入张量 `a` 和 `b` 必须为二维矩阵。
  * `a` 的第二个维度（列数）必须与 `b` 的第一个维度（行数）相等。
  * 输入张量的数据格式只支持ND。
  * 当输入矩阵的某些维度（如 `k` 或 `n`）不满足硬件内存对齐要求（例如512字节对齐）或超过特定阈值（如65536）时，算子内部会自动申请Workspace内存进行数据填充以优化性能。
